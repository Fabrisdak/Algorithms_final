# 0/1 Knapsack: Exact, Greedy, and ML-Based Solvers

This repository contains a reproducible implementation and evaluation of three approaches to the 0/1 Knapsack problem:

- Dynamic programming (exact baseline)
- Ratio-based greedy heuristic
- Supervised machine learning solvers using Random Forest and Multi-Layer Perceptron classifiers

The machine learning models are trained on labels generated by the exact dynamic programming solver. At evaluation time, predicted item selections are corrected for feasibility before computing objective value and optimality gap.

## Setup

Use Python 3.11 or newer. A clean virtual environment or Conda environment is recommended.

```bash
cd knapsack
pip install -r requirements.txt
```

## Reproducing the Pipeline

Run commands from the `knapsack/` directory.

### 1. Train the ML Models

The saved models are already included in `ml_pipeline/models/`. To retrain them:

```bash
python ml_pipeline/trainer.py
```

This regenerates:

- `ml_pipeline/models/rf_model.joblib`
- `ml_pipeline/models/mlp_model.joblib`

### 2. Run the Report-Round Experiments

```bash
python experiments/run_report_round.py
```

This evaluates DP, greedy, Random Forest, and MLP solvers across:

- Problem sizes: `n=20`, `n=50`, `n=100`
- Test families: uncorrelated, correlated, adversarial
- Instances per family and size: 20

The raw output is written to:

```text
experiments/results/report_round_results.csv
```

### 3. Regenerate Figures

```bash
python charts/generate_report_charts.py
```

This regenerates the figures in `charts/output/`:

- `report_gap_boxplots.png`
- `report_runtime_scaling.png`
- `report_gap_scaling.png`
- `report_gap_heatmap.png`

## Methods

The exact solver uses standard dynamic programming over item count and capacity, returning both the optimal objective value and selected item set.

The greedy solver sorts items by value-to-weight ratio and packs feasible items in that order. This is fast but not guaranteed to be optimal for the 0/1 problem.

The ML solvers treat each item as a binary classification decision. Features describe item value, weight, value-to-weight ratio, capacity context, and instance-level summary statistics. After prediction, a feasibility correction step removes selected items if needed so the returned solution respects capacity.

## Test Families

The experiments use three instance families:

- `uncorrelated`: values and weights are independently sampled.
- `correlated`: values are positively related to weights.
- `adversarial`: jackpot-and-bait instances designed to expose greedy and ML weaknesses.

## Results Snapshot

The included report-round CSV contains 720 solver evaluations. The main qualitative findings are:

- DP always achieves 0% optimality gap.
- Greedy is very strong on uncorrelated and correlated random instances, but is vulnerable to adversarial constructions.
- ML performance is sensitive to distribution shift because the models are trained on uncorrelated instances and tested on structurally different families.
- Random Forest is more robust than MLP on correlated instances, while both ML solvers struggle on adversarial instances.

See `charts/output/` for the generated visual summaries.
